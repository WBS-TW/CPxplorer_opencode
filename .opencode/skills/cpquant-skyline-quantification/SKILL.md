# CPquant Skyline Quantification

Use this skill for CPquant changes involving Skyline report import, standards, calibration, deconvolution, recovery/MDL calculations, isotope QC, or result downloads.

## Repo Triggers

- `R/CPquant.R`
- `R/CPquant_utils.R`
- `CPquant()`
- Skyline Excel input
- column alias normalization
- `Group Mixtures`
- calibration curves
- `perform_deconvolution()`
- isotope QC plots/tables
- `tests/testthat/test-cpquant-demo-data.R`
- `tests/testthat/helper-cpquant-fixtures.R`
- `inst/cpxplorer-demo_2026-04-01/`

## Repo Constraints

- Treat `Group Mixtures` as the supported standards mode unless the task explicitly expands the quantification model.
- CPquant expects Skyline Excel input and normalizes several possible Skyline column names up front.
- Preserve existing Skyline column aliases when changing import logic.
- Demo GC and LC files under `inst/cpxplorer-demo_2026-04-01/` are the closest integration fixtures.
- Downloaded Excel workbook sheets and column names are user-facing output contracts.

## Working Rules

- Keep import normalization near the existing import path.
- Validate both GC and LC demo-shaped data when touching import, calibration, or deconvolution logic.
- Preserve `Sample_Type` handling for `Standard`, `Unknown`, and `Blank` unless intentionally changing the data model.
- Keep isotope QC outputs consistent with `prepare_isotope_pattern_qc()` expectations.

## Verification

Preferred focused checks:

- `devtools::test(filter = "cpquant-demo-data")`
- `devtools::load_all(); CPquant()`

When download output changes, inspect workbook sheet names and key columns generated by `output$downloadResults`.

## Completion Checklist

- Preserved Skyline column aliases or documented intentional changes
- Checked GC and LC demo fixture coverage where relevant
- Ran or attempted CPquant focused tests
- Verified user-facing workbook/table output shape when changed
