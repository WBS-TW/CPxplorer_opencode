# Roxygen README Sync

Use this skill when documentation or generated package metadata is part of the change.

## Repo Triggers

Common triggers in this repo:
- `README.Rmd`
- `README.md`
- roxygen comments in `R/*.R`
- `NAMESPACE`
- `man/*.Rd`
- requests to update package docs or exported help pages

## Source Of Truth

- `README.md` is generated from `README.Rmd`.
- `NAMESPACE` is generated by `roxygen2`.
- `man/*.Rd` files are generated by `roxygen2`.

## Working Rules

- Edit `README.Rmd`, not `README.md`, unless the task explicitly requires a generated-file-only adjustment and the repo conventions allow it.
- Edit roxygen comments in `R/*.R`, not `NAMESPACE` or `man/*.Rd`.
- Keep examples and docs aligned with current exported behavior.
- Avoid partial manual updates that leave generated files stale.

## Regeneration

When roxygen docs changed:
- run `devtools::document()`

When README changed:
- re-knit `README.Rmd` to regenerate `README.md`

## Verification

- Confirm generated files reflect the intended source edits.
- If documentation references runtime behavior, sanity check the relevant function or app entrypoint.
- Mention if regeneration was not run.

## Completion Checklist

- Source documentation files updated
- Generated artifacts refreshed when appropriate
- Final summary clearly distinguishes edited source files from regenerated files
