# CPxplorer Agent Notes

## Repo Shape
- This repo is a single R package, not a multi-package workspace. The exported entrypoints are `CPions()` and `CPquant()` in `R/CPions.R` and `R/CPquant.R`.
- Most app logic lives directly inside those two large Shiny files; shared helpers are in `R/CPions_utils.R` and `R/CPquant_utils.R`.
- Runtime assets are loaded from `inst/` via `system.file(...)`. In particular, the apps render `inst/instructions_CPions.md` and `inst/instructions_CPquant.md` at runtime.
- Demo/sample files live under `inst/cpxplorer-demo_2026-04-01/` and are the closest thing to integration fixtures.

## Generated Files
- `README.md` is generated from `README.Rmd` (`README.Rmd` says "Knit this README.Rmd to generate README.md"). Edit the `.Rmd`, then re-knit.
- `NAMESPACE` and `man/*.Rd` are generated by `roxygen2` (`NAMESPACE` says "do not edit by hand", and `DESCRIPTION` enables roxygen markdown). Edit roxygen comments in `R/*.R`, then regenerate docs.

## Verification
- The repo has a `tests/testthat` suite with CPions snapshots, CPions Shiny server smoke tests, and CPquant demo-data tests. Prefer focused `devtools::test(filter = "...")` runs for changed areas.
- App launch verification is still useful for interactive Shiny changes, e.g. `devtools::load_all(); CPions()` or `devtools::load_all(); CPquant()`.
- For package-level verification after roxygen or metadata changes, use normal R package checks (`devtools::document()`, then build/check as needed).

## Repo-Specific Constraints
- `CPquant()` currently only exposes one standards mode: `Group Mixtures`. Treat that as the supported path unless you are intentionally expanding the quantification model.
- `CPquant()` expects Skyline Excel input and normalizes several possible Skyline column names up front; changes to import logic should preserve those aliases.
- `CPions` interference and Skyline export flows depend on state computed in earlier tabs (`CP_allions_compl2` is populated by the interference step). If you change tab behavior, verify the cross-tab flow still works.
- `R/CPions_utils.R` contains a large embedded `isotopes` table used by isotope pattern generation. Avoid editing or duplicating it unless the mass table itself is the intended change.

## Instruction File Maintenance
- `inst/instructions_CPions.md` and `inst/instructions_CPquant.md` are rendered in-app via `includeMarkdown` (`R/CPions.R:190`, `R/CPquant.R:229`). Whenever UI features are added, removed, or changed in either app file, update the corresponding instruction file to inform the user. Keep entries short and informative.
- `inst/instructions_CPions.html` is a legacy static HTML copy — update it if the `.md` changes and the HTML version is still in use.
